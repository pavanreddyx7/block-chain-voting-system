Blockchain Voting Project

This updated project removes the fingerprint sensor completely. The Arduino Mega handles push-button voting and LCD display, while the ESP8266 NodeMCU sends the confirmed vote to a backend and blockchain network over Wi-Fi.

No fingerprint module Arduino Mega controller NodeMCU ESP8266 communication Push-button voting 16x2 LCD status display

Project Working

1

System ready

LCD shows Ready to vote. Mega waits for button input and NodeMCU keeps Wi-Fi connected.
2

Candidate select

Voter presses one of the candidate buttons. Mega updates the LCD with the selected candidate name.
3

Vote confirm

The confirm button sends a compact serial message from Mega to NodeMCU, such as VOTE:A.
4

Upload to backend

NodeMCU performs an HTTP request to your server. The server checks duplicate voting and then stores or forwards the vote to blockchain.
5

Result on LCD

NodeMCU replies with SUCCESS or FAIL. Mega shows Vote stored or Send failed on the LCD.

Main Modules

Arduino Mega

Reads all buttons, manages LCD, controls voting flow.

NodeMCU ESP8266

Connects to Wi-Fi and sends vote data to API/server.

16x2 LCD

Displays ready state, selected candidate, and final message.

Push Buttons

Candidate A, Candidate B, Candidate C, and confirm input.
Suggested demo buttons: A, B, C, and CONFIRM.

Button C D24 Use INPUT_PULLUP, button to GND
Confirm button D25 Sends final selected vote
LCD SDA Pin 20 I2C data on Mega
LCD SCL Pin 21 I2C clock on Mega
NodeMCU TX RX1 pin 19 Serial from NodeMCU to Mega
NodeMCU RX TX1 pin 18 Use voltage divider for Mega TX to ESP RX

Power and Safety Notes

1
Power the Mega and LCD from a stable 5V supply.
2
Power the NodeMCU from USB or a regulated 3.3V capable source.
3
Do not connect Mega 5V TX directly to NodeMCU RX. Use a voltage divider or logic level shifter.
4
All modules must share one common ground connection.
